Os princípios SOLID nos dizem como organizar as funções e estruturas de dados em classes e como essas classes devem ser interconectadas. O uso da palavra "classe" não implica que esses princípios sejam aplicáveis apenas a software orientados a objetos. Uma classe é apenas um agrupamento acoplado de funções e dados. Cada sistema de software tem agrupamentos como esses, chamados ou não de classes. Os princípios SOLID se aplicam a esses agrupamentos.

SOLID é um acrónimo de 5 princípios que tornam software mais:
 - Manutenível - fácil de modificar
 - Extensível - fácil de adicionar features
 - Testável - fácil de escrever testes
 - Compreensível - fácil de entender

Temos os princípios:
- **SRP: Princípio de Responsabilidade Única** ([[01 Single Responsibility Principle (SRP)]])
- **OCP:** Princípio Aberto/Fechado ([[02 Open-Closed Principle (OCP)]])
- **LSP**: Princípio de Substituição Liskov ([[03 Liskov Substitution Principle (LSP)]])
- **ISP:** Princípio de Segregação de Interface ([[04 Interface Segregation Principle (ISP)]])
- **DIP:** Princípio de Inversão de Dependência ([[05 Dependency Inversion Principle (DIP)]])
